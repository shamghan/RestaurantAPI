# RestaurantAPI

Clean .NET 8 APIs built with Clean Architecture, featuring user Identity (JWT auth \& role-based access) and Azure deployment via CI/CD. Includes practical exercises for CRUD, secure endpoints, EF Core, and automated workflows for scalable, maintainable API development.





.NET 8 Clean Architecture API with Identity \& Azure CI/CD



This project demonstrates building scalable and maintainable APIs using .NET 8 following Clean Architecture principles. It includes a full user identity system, secure authentication and authorization, and deployment to Microsoft Azure using a CI/CD pipeline.



Key Features

Clean Architecture – Separation of concerns with layers for API, Application, Domain, and Infrastructure.

User Identity System – Implements registration, login, role-based authorization, and JWT authentication.

.NET 8 APIs – Modern, high-performance APIs with proper RESTful design.

Azure Deployment – Automatic deployment using GitHub Actions / Azure DevOps CI/CD pipelines.

Practical Exercises – Includes hands-on examples for:

CRUD operations with EF Core

Secure API endpoints

Role-based access control

Environment-based configuration

Logging and exception handling

Technologies

.NET 8

Entity Framework Core

SQL Server / Azure SQL

Azure App Services

GitHub Actions / Azure DevOps

JWT / Identity

Getting Started

Clone the repository.

Configure the database connection in appsettings.json.

Run migrations using dotnet ef database update.

Start the API and test endpoints using Swagger or Postman.

Explore the Azure CI/CD workflow for automatic deployment.
